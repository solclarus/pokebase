# Pokemon Data API

Pokémon data API for mainline series and Pokémon GO.

## Structure

```
pokemon/
├── apps/
│   ├── api/          # REST API — Cloudflare Workers + Hono
│   └── web/          # Dashboard — Next.js App Router
├── data/             # JSON data files (OSS)
└── packages/
    ├── schemas/      # Zod schemas (shared between API and Web)
    └── scripts/      # Data validation and index generation
```

## Setup

```bash
vp install
```

## Development

```bash
# API (Cloudflare Workers local)
vp run dev

# Web (port 3000)
vp run dev:web
```

Add the API URL to `apps/web/.env.local`:

```
API_URL=http://localhost:8787
```

## Commands

| Command              | Description                |
| -------------------- | -------------------------- |
| `vp run dev`         | Start API local server     |
| `vp run dev:web`     | Start Web local server     |
| `vp run validate`    | Validate data (Zod)        |
| `vp run build:index` | Regenerate `data/_index/`  |
| `vp run deploy`      | Deploy API to Cloudflare   |
| `vp run deploy:web`  | Deploy Web to Cloudflare   |
| `vp check`           | Format + lint + type check |

## Data Structure

```
data/
├── _index/           # Auto-generated by CI — lightweight index
├── core/
│   ├── pokemons/     # Base species info (one file per Pokémon)
│   ├── forms/        # All forms + stats
│   └── abilities/    # Abilities
├── mainline/
│   ├── games.json    # Game list
│   ├── moves/        # Moves
│   ├── learnsets/    # Learnsets
│   └── availability/ # Per-game availability
└── go/
    ├── forms/        # GO form stats and movesets
    ├── costumes/     # GO-exclusive costumes
    └── moves/        # GO moves
```

Filenames are zero-padded to 4 digits (e.g. `0006.json`).

## API Endpoints

```
GET /pokemon                  # List
GET /pokemon/:id              # Detail (Pokédex number or identifier)
GET /pokemon/:id/moves        # Learnable moves
GET /pokemon/:id/availability # Game availability
GET /moves                    # Move list
GET /moves/:id                # Move detail
GET /games                    # Game list
GET /games/:id/pokemon        # Pokémon available in a game
GET /abilities                # Ability list
GET /abilities/:id            # Ability detail
GET /go/pokemon               # GO Pokémon list
GET /go/pokemon/:id           # GO Pokémon detail
GET /go/pokemon/:id/costumes  # GO costumes
GET /go/moves                 # GO move list
GET /go/moves/:id             # GO move detail
```

API docs (Scalar UI): `/doc`

## CI/CD

| Workflow         | Trigger                                                   | Description                                            |
| ---------------- | --------------------------------------------------------- | ------------------------------------------------------ |
| `validate.yml`   | PR (`data/`, `packages/scripts/` changes)                 | Data validation + index diff check                     |
| `deploy-api.yml` | Push to `main`                                            | Regenerate index → deploy to Cloudflare Workers        |
| `deploy-web.yml` | Push to `main` (`apps/web/`, `packages/schemas/` changes) | Build schemas → build Web → deploy to Cloudflare Pages |

## Deployment

Required secrets (GitHub Actions):

| Secret                 | Description          |
| ---------------------- | -------------------- |
| `CLOUDFLARE_API_TOKEN` | Cloudflare API token |

Cloudflare Workers environment variables (`wrangler.toml` [vars]):

| Variable          | Description                      |
| ----------------- | -------------------------------- |
| `ALLOWED_ORIGIN`  | Allowed CORS origin (Web domain) |
| `API_URL`         | Production API URL (SSR)         |
| `IMAGES_BASE_URL` | Image CDN base URL               |
